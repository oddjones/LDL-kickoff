The LDL-Kickoff codekit project has been set up to automatically compile all .KIT files into the root app folder. By default, we have placed all our KIT files in a "kits" folder, within our main BUILD folder. It has also been set up to ignore the "partials" subfolder - so any .KIT files placed in that folder will not be compiled. They will however, be included into any files which reference them. This allows us to build a kit of parts in the "kits/partials" folder and use them to build up pages quickly and simply.

As you build, codekit will compile on the fly, saving your .HTML files into the /app folder.

How do I get started?

Take a look at the code making up the homepage of the default app in LDL-Kickoff - it is almost entirely constructed out of "kit parts". 

      <!-- @include "partials/header" -->
      <div id="pageContent" class="container homePage">
        <div id="content" class="row">
          <div class="col-sm-8">
            <!--@include "/partials/carousel"-->
          </div>
          <div class="col-sm-4">
          <!--@include "/partials/RightColumn-Asides/related-stuff"-->
          </div>
        </div>
        <br>
        <!--@include "/partials/landing-list"-->
        <!--@include "/partials/landing-more-list"--> 
      </div>
      <!-- @include "partials/footer" -->
Some of the inlude files are themselves made up out of include files - take the partials/header for example, which references both the search and nav/topnav KIT files.
