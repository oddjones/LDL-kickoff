Working with .KIT files

.KIT files are Codekit's way to implement .ASP style includes. KIT files are complied by Codekit into .HTML files located in the root of the .app folder of your compiled application. KIT files form the basis of LDL-Kickoff's modular construction. You can include one KIT file into another simply by using the <!-- @include "filename" --> directive. You can nest kits within kits within kits if you like.

How does it work?

The LDL-Kickoff codekit project has been set up to automatically compile all .KIT files into the root app folder. By default, we have placed all our KIT files in a "kits" folder, within our main BUILD folder. It has also been set up to ignore the "partials" subfolder - so any .KIT files placed in that folder will not be compiled. This allows us to build a kit of parts in the "kits/partials" folder and use them to build up pages quickly and simply.

How do I get started?

Take a look at the code making up the homepage of the default app in LDL-Kickoff - it is almost entirely constructed out of "kit parts". 

      <!-- @include "partials/header" -->
      <div id="pageContent" class="container homePage">
        <div id="content" class="row">
          <div class="col-sm-8">
            <!--@include "/partials/carousel"-->
          </div>
          <div class="col-sm-4">
          <!--@include "/partials/RightColumn-Asides/related-stuff"-->
          </div>
        </div>
        <br>
        <!--@include "/partials/landing-list"-->
        <!--@include "/partials/landing-more-list"--> 
      </div>
      <!-- @include "partials/footer" -->
Some of the inlude files are themselves made up out of include files - take the partials/header for example, which references both the search and nav/topnav KIT files.

Where do I start?

Build your HTML layouts in the build folder. Take the index.kit file for a start and open up its component parts. Either edit or make copies of the components and replace them with sections pertinent to your own site. We have provided the mockup and construction folders for you to dump Photoshop/ Balsamiq/ Saved HTML startup code into - these folders mean the project has its whole history saved together. To that end, there's even a project folder for pre-reqs, PIDs, spider documents etc. The tests folder is for handing over to dev/ops (or at least it will be when I can think of something to put in there).

REMEMBER!

Only the compiled /app folder will be handed over to developers, so don't worry about making a mess in the build folder, that's your folder to do with as you please!
