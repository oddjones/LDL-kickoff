Compiling SASS (.scss) Files

At release 0.01 LDL-Kickoff has its own set of mixins which build on top of Bootstrap's built in collection. This allows us to simply include a compiled stylesheet for Bootstrap without relying on a specific version (we still have a number of sites which either don't use bootstrap or use version 2.x). It's up for discussion whether we start utilising Bootstrap's variables.scss and mixins Whilst Codekit will support SASS libraries and frameworks such as Compass and Bourbon it's considered a good idea to "walk before we run" - there's no reason why we shouldn't adopt these later on, but it's best to get comfortable with the basic concepts first.

How does it work?

Much in the same way as with the project Javascript files, LDL-Kickoff compiles a number of different library, framework, app specific and LDL global files together into a single .CSS file (/app/css/app.min.css). The file which does all the work is /build/scss/app.scss. The easiest way to see how it works is to look at the code of this file. Notice that it uses @import statements to include all the separate scss files it is referencing. They are listed in the table below with their usage

filecontents
librariesImports all the library files (bootstrap, font-awesome etc)
globals/colorsA list of custom defined colors - these are illustrated on the "color-swatches.png" - this also includes global LCC/LDL colours
globals/custom-mixinsgradients, rounded corners, circles, shadows, global variables etc (add your own!)
globals/fontsused for defining custom fonts
globals/defaultsDefines some resets. There are also some LDL/LCC specific global elements defined here such as
  • landing-list: Landing page thumbnail elements
  • more-links: Liverpool.gov landing page "more" list
  • listPageList: "News Item" style large list
  • imgHolder: placeholder for inline img with caption
partials/viewportApp/Site specific styles - these may be branding such as Connect2
partials/...Any repeatable sections - these will be .kit files (such as right column wells etc) which have dependent styles
globals/mobileMobile specific
